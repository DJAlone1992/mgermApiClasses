# /appointmentCallHistory.php

Точка входа для изменения данных о статусе подтверждения направления пациентом
## Параметры

Метод: POST
Параметры:
  - recordID (int) - Номер записи
  - action (enum:string) - Вид взаимодействия:
    - CONFIRM - Пациент подтвердил прием
    - DENIED - Пациент отказался от приема
### Ответ
Успешная смена статуса HTTP 200
```json
{
    "processed": 1
}
```
Ошибка HTTP 500
```json
{
    "message":<Описание типа ошибки при создании>,
    "error":<Код ошибки при создании>,
}
```
